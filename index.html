Synthetic Object Detection training Using GIS Data and Unity Engine


The Problem

Remote Sensing Data with high geospatial
resolution are very hard and time consuming to find
Their access may be prohibited due to safety and privacy concerns
The sensor and equipment to capture them might not be available to an organization

The Solution

By utilizing GIS Data and the Unity Game Engine
digital simulations can be created that represent a specific real world area
where objects can be placed, photographed and automatically labeled.


Data Sources/Processing

Copernicus

* Sentinel-2 product representing the area of Rhodes

* It's NDVI and SAVI index was extracted

* Useful for more accurate representation of the flora at a given time

OpenStreetMap

* Building Footprint and Road Network data were downloaded

* The data were converted in a .shp format

ArcGIS Pro Basemap

* All the aforementioned data were inputed

in the ArcGIS Pro basemap

* Data Processing and Cleaning


Data Fusion

All the aforementioned data were inputed in the ArcGIS Pro basemap

For a more precise building detection, CNN algorithm was used in the Basemap image

The building data were fused and cleaned so that double data are non existent and they are within the basemap limit

Random Forest algorithm to cluster the image with fertile ground, low vegetation, and high vegetation

Heighmap to Simulation

The final product of the Data Fusion step is converted to a greyscale image.

This image is used as a Heighmap in the Unity Engine, each pixel value represents in Unity the height of the terrain.

Because each class has a specific value (thus a specific height), the object that corresponds to each class is placed using C# Scripting

The terrain is then levelled providing a visualization of the area of interest.

Synthetic Image Generation

Now that the AOI simulation is ready objects(cars) are placed at random locations.

Using a C# Script a bounding box coordinates are calculated and are converted from Unity to screen coordinates.

Finally the screen coordinates are converted to YOLO coordinates as this is the algorithm that was used for Object Detection.

The algorithm is trained on the generated synthetic images and is tested on real images of the AOI.
